# 대입·유학 사전 자가점검 키트

학원·유학원 **유료 컨설팅을 받기 전에** 정량 스펙 기반으로 본인 위치를 빠르게 확인하는 0단계 스크리닝 도구.

## 목적과 신뢰 범위 (중요)

> **포지션:** "합격을 예측하는 컨설팅"이 아니라, **"컨설팅 받기 전에 내 위치를 정확히 알고 가게 해주는 0단계 도구"**.
> 이 선을 지키면 신뢰할 수 있고, 넘으면(합격 보장처럼 말하면) 대형학원만 못한 가짜가 된다.

### ✅ 잘하는 것 (학원 1차 상담 수준)
- **정량 위치 스크리닝** — 내 모의 백분위가 목표 대학 작년 70%컷 대비 어느 라인인지 즉답
- **정보 비대칭 해소** — 상담 가기 전 내 위치·지원 라인·따질 질문을 미리 파악 → 휘둘리지 않게
- **무료·즉시·무제한** 반복 점검
- **정직한 한계 표기** — 입결 데이터 없으면 가짜 판정 대신 "보류", 추정치는 추정치로 명시

### ❌ 못하는 것 (대형학원 컨설팅 대체 불가)
- **합불 확률 정밀 예측** — 학원의 다년치 합불 누적("칸") 데이터가 없음
- **표준점수 환산** — 현재 백분위 기반(변별력은 표준점수보다 낮음)
- **대학별 실제 환산식** — 반영비율은 계열 표준 **추정치**(모집요강 실제값 아님)
- **수시 정성평가** — 에세이·비교과·면접 미반영
- **실시간 지원동향** — 작년(2025) 정적 데이터 기반, 올해 눈치·펑크 예측 불가
- **개인 맞춤 종합 전략** — 룰 기반 일반 조언

> 학원급에 *근접*하려면: 표준점수 도입 → 대학별 실제 환산식 적재 → 수시 내신 모드 → 다년치 추세 보정.

## 폴더 구조

```
university/
├── 기본sop.md              # 프로젝트 헌법 (페르소나·규칙)
├── backend/                # Spring Boot 3 + JPA (H2 dev / MySQL prod)
├── frontend/               # React + Vite + Tailwind + Recharts
└── docs/
    ├── 20260519-1500.md    # 초기 빌드 Why 핸드오프
    └── SECURITY_HANDOFF.md # 잔존 보안 리스크
```

## 빠른 실행

```bash
# 1) 백엔드 (H2 인메모리, 추가 설치 불필요)
cd backend
./gradlew bootRun       # Windows: .\gradlew.bat bootRun

# 2) 프론트엔드
cd frontend
npm install
npm run dev
# → http://localhost:5173
```
