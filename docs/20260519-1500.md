# 핸드오프 — 초기 빌드 (2026-05-19 15:00)

## 결정 사항 (PM 승인 완료)

기획서가 명시하지 않았으나 코드화에 필수적이었던 5개 항목을 PM 제안 기본값대로 확정.

### 1. SAT 환산 공식
- **공식:** `SAT = 400 + (국어백분위 + 수학백분위) / 200 × 1200`, 10점 단위 반올림.
- **Why:** 기획서가 "정규화 변환"만 언급. 백분위 합 0이면 SAT 하한 400, 합 200(만점)이면 1600에 맵핑되어 SAT 공식 스케일(400~1600)과 정확히 일치. 단순·역추적 가능·검증 가능하다는 3박자가 0단계 스크리닝 도구 철학과 부합.
- **How to apply:** `ScoreConversionService.percentileSumToSat()` 단일 진입점. 향후 영어 등 추가 시 이 메서드만 확장.

### 2. 신호등 임계값
- **국내:** 가중 백분위 - 컷오프 격차 기준. ≥ -5pp → 안정 / -5~-15pp → 소신 / < -15pp → 상향.
- **해외:** SAT/GPA 분포 내 위치비율 평균. ≥ 0.55 → 안정 / 0.30~0.55 → 소신 / < 0.30 → 상향.
- **확률 범위:** 안정 60~80%, 소신 40~60%, 상향 10~40%.
- **Why:** "범위(40~60%)"로 표현하는 것은 학원 공포 마케팅 방어가 본 도구 목적이기 때문 — 단일 숫자로 줄이면 도리어 새 미신을 만든다. 격차를 -5/-15pp로 끊은 이유는 정시 컷이 해마다 ±5pp 내외로 출렁이기 때문, 그 노이즈 폭이 안정/소신을 가르는 자연 경계.
- **How to apply:** Service 내부 `SignalBucket` enum. 추후 데이터 누적되면 분위 기반(분포 백분위)으로 교체 예정.

### 3. 탐구 가중치 처리
- 탐구1·2 백분위를 단순 평균한 뒤 `ratio_tg` 곱.
- **Why:** 대다수 정시 환산이 두 과목 평균을 그대로 쓴다(서울대 일부 예외). 초기 스크리닝 단계에서 학교별 변종까지 분기하면 입력 폼이 무거워진다.
- **How to apply:** `DomesticDiagnosticService.averageTg()`. 한쪽만 입력 시 그 값을 그대로 사용.

### 4. 합격 확률 문구의 톤
- "X% ~ Y%" 범위 표현 + "본 결과는 정량 스펙 기반 사전 스크리닝" 푸터 고정.
- **Why:** 단일 확률("하버드 50%")로 표기 시 만능 예측기로 오인될 법적·윤리적 리스크. 기획 철학("만능 예측기 ❌")과 정면 충돌. 범위 + 디스클레이머로 통계적 위치임을 명시.
- **How to apply:** `DiagnosticResult.jsx` 푸터 + 신호등 칩에 % 범위 병기.

### 5. DB 시드 데이터 소싱
- 1차: CSV 시드(`src/main/resources/data/*.csv`)로 H2/MySQL 양쪽에 부팅 시 로드.
- 2차: 추후 대교협·CDS 크롤러는 별도 PR.
- **Why:** SOP 3항 "외부 오픈소스로 자동 채움" 원칙에 부합하되, 0단계 MVP에서 크롤러 검증에 시간을 쓰면 본 가설(사전 스크리닝 도구의 유용성) 검증이 늦어진다. 일단 정적 시드로 동작시키고 사용자 반응을 본다.
- **How to apply:** `DataInitializer.java`가 OpenCSV로 부팅 시 1회 적재. 빈 테이블만 채워서 멱등.

## 아키텍처 한눈에

```
[Frontend: Vite+React+Tailwind+Recharts]
   ServiceModeToggle ──┐
   DiagnosticInputForm ─┼─ POST /api/diagnostic ──> [Backend: Spring Boot]
   DiagnosticResult    ─┘                              │
                                                       ├─ DomesticDiagnosticService (Z·가중·신호등)
                                                       ├─ GlobalDiagnosticService  (GPA·SAT 환산)
                                                       └─ ScoreConversionService   (공통 환산 코어)
                                                              │
                                                       [H2 (dev) / MySQL (prod)]
                                                              │
                                                       DataInitializer  ← CSV 시드 (Fallback 캐시 겸용)
```

## 실행 방법

```bash
# 백엔드 (H2 인메모리, 별도 DB 설치 불필요)
cd backend
./gradlew bootRun
# → http://localhost:8080/api/diagnostic/health → "ok"

# 프론트엔드
cd frontend
npm install
npm run dev
# → http://localhost:5173
```

MySQL 전환은 `backend/.env`에 DB_URL 등 채우고 `SPRING_PROFILES_ACTIVE=mysql ./gradlew bootRun`.
