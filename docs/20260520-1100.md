### 2. 고교 유형 기반 "수시 환경" 보조 진단 ✅
- **핵심 결정 (Why 정시 백분위를 보정하지 않는가):** 현재 진단 입력은 모의고사/수능 등급 = 전국 상대평가다. 대형학원도 정시 합격예측에서는 고교 유형 보정을 하지 않는다(특목고생·일반고생의 같은 모의 등급은 같은 의미). 학원이 고교 유형 보정을 적용하는 곳은 **수시 학생부(내신)**다.
- **따라서:** 정시 백분위(`nationalPercentile`)는 불변. 출신 고교 유형(NEIS `HS_SC_NM`)에 따라 **수시 전략 안내 + 동적 압박질문**만 추가.
- **변경:**
  - `HighSchool` 엔티티 +`highSchoolType` (NEIS `HS_SC_NM`: 일반고/특수목적고/특성화고/자율고). **기존 `schoolKind`는 `SCHUL_KND_SC_NM`=전부 "고등학교"라 유형 구분 불가였음** → 신규 필드 필수였다.
  - `NeisSyncService` 매핑에 `HS_SC_NM` 적재 추가(생성·갱신 양쪽).
  - 신규 `HighSchoolContextService` — 유형별(특목고/자사고/자율고/특성화고/일반고) 학원식 수시 안내 메시지 + 추가 질문 생성. 자사고는 `자율`+`사립` 조합으로 판별.
  - `DomesticDiagnosticService` — 고교 코드(우선) 또는 명칭으로 조회 → context 병합, 응답에 `highSchoolType`/`highSchoolContext` 추가.
  - `DiagnosticResponse` +`highSchoolType` +`highSchoolContext`
  - `DiagnosticResult.jsx` "🏫 수시 환경 안내" 블록 추가(국내·context 있을 때만).

## 추가 수정 (실행 검증 중 발견·해결)
### A. `.env` 자동 로딩 추가 — `spring-dotenv`
- **문제:** Spring Boot는 `.env`를 기본으로 읽지 않는다. `.env`에 NEIS/KCUE 키를 넣어도 `${NEIS_API_KEY:}`가 빈 값으로 해석돼 키가 무시됐다.
- **해결:** `build.gradle`에 `me.paulschwarz:spring-dotenv:4.0.0` 추가. bootRun 작업디렉터리(`backend/`)의 `.env`를 Spring Environment로 자동 주입.

### B. NEIS 동기화 `OK_0` 버그 — WebClient 버퍼 한도
- **증상:** NEIS 키 정상·API 응답 정상인데 적재 0건(`OK_0`).
- **원인:** `pSize=1000` 응답이 수백 KB~MB인데 WebClient 기본 in-memory 버퍼가 256KB라 디코딩 실패 → `onErrorResume`이 조용히 빈 결과로 처리.
- **해결:** `NeisSyncService`/`KcueSyncService`/`UsaScorecardSyncService` 세 곳 모두 `.codecs(c -> c.defaultCodecs().maxInMemorySize(16MB))` 적용.
- **검증:** 재기동 후 `neis.lastStatus = OK_2408`. 고교 검색·진단 수시환경 안내 end-to-end 정상.
- **참고:** `kcue.lastStatus = FALLBACK_KEPT`(에러→폴백 CSV 사용 중). 국내 대학 컷은 시드/스테이징으로 동작. KCUE 실 응답 매핑 점검은 별도 과제(SECURITY_HANDOFF 13).

## M6 — 신호 체계 학원식 정합화 + 해외 게이지 수정 (실사용 스크린샷 피드백 반영)
실제 결과 화면에서 두 가지 표현 오류 확인 → PM 권한으로 학원 방식에 맞춰 일괄 수정.

### C. 지원가능성 신호 3단계 → 학원식 4단계
- **문제(국내):** 가중 백분위가 목표 70%컷보다 낮은데(−3.2pp) "안정"으로 표기. 기존 임계가 `gap >= -5 → 안정`이라 컷 미만도 안정 처리.
- **결정:** 공통 `service/Signal.java` 신설, 4단계 **안정/적정/소신/위험**(학원 관례).
  - 국내(`ofDomesticGap`): gap≥0 안정 / ≥−4 적정 / ≥−10 소신 / 그 미만 위험.
  - 해외(`ofGlobalComposite`): ≥0.60 안정 / ≥0.40 적정 / ≥0.20 소신 / 그 미만 위험.
  - 확률밴드: 안정 75-95 / 적정 55-75 / 소신 30-55 / 위험 5-30.
- **Why 임계값:** 70%컷은 합격자 70% 안정선 → 컷 미만은 안정 불가가 핵심 제약. 정확한 칸 보정은 누적 합불 데이터 확보 후 재캘리브레이션(주석 명시).
- `DomesticDiagnosticService`/`GlobalDiagnosticService` 내부 SignalBucket enum 제거 → 공통 `Signal` 사용.

### D. 해외 게이지 = 한국 백분위 → 목표교 대비 위치
- **문제(해외):** 게이지가 한국 전국 백분위(89%)를 표시하면서 판정(상향)과 모순. 한국 백분위는 해외 진학과 무관.
- **해결:** `GlobalDiagnosticService.nationalPercentile = composite*100`(목표교 CDS 대비 종합 위치). 죽은 코드 `bestKoreanSubjectGrade`(이름과 달리 평균 반환) → `averageMainGrade`로 정리.
- **검증:** Columbia 케이스 게이지 0.0 + "위험"으로 일관(SAT 1470=CDS 바닥, GPA 3.70<평균 4.12).

### E. 결과 UI (`DiagnosticResult.jsx`)
- `SIGNAL_STYLE` 4색(안정 green / 적정 blue / 소신 amber / 위험 red).
- 게이지 라벨 모드별: 국내 "전국 백분위 + 상위 N%", 해외 "목표교 대비 위치".
- 해외 GPA/SAT에 "등급 기반 추정치 — 유학원 공식과 다를 수 있음" 주의 문구.

### 검증 결과(M6)
- compileJava BUILD SUCCESSFUL, bootRun 정상.
- 국내 한양대 경영(−3.2pp): "적정"(55-75%) — 기존 오판 "안정" 교정 확인.

## M8 — "(미등록 대학)" 근절 + 데이터 출처 진실 규명 (사용자: "주요대학은 미등록 없어야 신뢰도 ↑")
### 근본 원인 (2중)
1. **자동완성 ≠ 진단 소스:** 자동완성은 `university_programs`(대학알리미 편제단위 xlsx, 컷 없음)에서, 진단은 `domestic_university_cutoffs`(20줄 시드)에서 조회 → 자동완성으로 고른 대다수가 입결표에 없어 fallback.
2. **가짜 판정:** 미매칭 시 `defaultCutoff()`가 "(미등록 대학)" + 임의 컷 80 → 거짓 "안정" 표출.

### 데이터 출처 진실 (중요)
- 어제 등록한 키는 **대학알리미(academyinfo.go.kr)**. 직접 호출 결과 `/univCutoff`는 **에러 페이지**(존재하지 않는 엔드포인트). KcueSyncService의 `/univCutoff`·`cutoffPercentile70` 등은 전부 잘못된 가정이었음(SECURITY_HANDOFF 13 현실화).
- **대학알리미는 정시 70%컷 입결을 공시하지 않는다.** 공시 항목은 학과정보·등록금·충원율·취업률 등. → 대학알리미 API로는 입결을 영원히 못 채움.
- **정시 70%컷 입결은 같은 대교협의 '어디가(adiga.kr)'에 있다.** 단, 어디가는 OpenAPI가 아니라 포털 다운로드(엑셀/PDF) 형태. → staging CSV 파이프라인으로 적재해야 함.

### 수정 (가짜 제거 = 정직 우선)
- 자동완성 소스: `programRepo` → `domesticRepo`(진단 가능한 입결표만 노출). 선택 시 미등록 발생 안 함.
- `DiagnosticResponse` +`cutoffAvailable`. 입결 없으면(미매칭 or 컷 null) 가짜 컷 판정 대신 `cutoffAvailable=false` + 전국 백분위(기본 반영비율) + 일반 조언. 입력한 대학명 그대로 표기.
- `defaultCutoff()` 삭제. `SolutionService.buildDomesticNoData` 추가.
- `DiagnosticResult.jsx`: 입결 없음 시 신호배지 숨김 + 황색 안내 배너.
- **검증:** 서울대 경제학부(시드 미존재) → cutoffAvailable=false, 가짜 안정 사라짐. 한양대 경영(시드 존재) → 정상.

### ⛔ 시드 임의 확장은 보류 (PM 판단)
- 주요대학 입결을 코드로 지어내면 [[follow-big-academy-methods]] 위반 + 신뢰도 훼손. **실제 어디가 입결을 staging에 적재하는 것이 정답.** 출처 결정은 사용자 몫(아래 후보 참조).
