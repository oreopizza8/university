## 변경 요약

### 1. 시크릿 인시던트 처리

- 즉시 조치:

  - `backend/.env.example` 은 placeholder 로 복원.
  - `.gitignore` 에 `backend/.env`, `frontend/.env` 명시 추가.
- **Why:** SOP 1항 "하드코딩 금지" 직접 위반. 푸시 전 차단이 핵심 — 이미 푸시됐다면 키 폐기 필요. `SECURITY_HANDOFF.md` 3항에 인시던트 기록.

### 2. 대학알리미 학과 마스터 통합
사용자가 다운받은 `학교별 교육편제단위 정보_20241007기준.xlsx` 정체 = **대학알리미 학과 마스터 55,233행 / 23컬럼**.

흐름:
1. `backend/data-staging/_convert_xlsx_to_csv.py` — Python 1회 변환기. 한글 헤더 → 영문, 폐지/모집중단 학과 제외 후 UTF-8 CSV 출력.
2. 변환 결과: **활성 학과 28,104건** (폐지 27,124건 제외).
3. `UniversityProgram` 엔티티 + Repository 신규.
4. `StagingDataLoader.loadPrograms()` — 부팅 시 1,000건 단위 배치 적재.
5. `/api/diagnostic/universities/domestic` 자동완성을 `UniversityProgram` 기반으로 교체. 더 풍부한 메타(지역·계열·학교코드) 응답.

**Why Python 변환기 (vs Apache POI 영구 의존):**
- POI 추가 시 JAR 크기 5~10MB + 빌드 시간 증가.
- xlsx 갱신 주기는 분기/연 단위 → 영구 의존 가치 낮음.
- 사용자 환경에 이미 Python 3.14 + openpyxl 가용.
- 변환기는 staging 폴더에 1회용으로 남겨두고, Java 는 깨끗한 CSV만 처리.

**Why 폐지·모집중단 사전 필터:**
- 자동완성에 폐지 학과가 섞이면 사용자 혼란.
- 데이터 크기 절반 감소(55k → 28k) → 인메모리 H2 부팅 시간·메모리 절약.
