# 핸드오프 — M4 결정 반영 + 해외 크롤러 + 분포 강화 (2026-05-19 20:00)

## PM 결정 3건 반영 결과

### 1. NEIS = 고등학교 마스터 한 가지로 좁힘 ✅
- `HighSchool` 엔티티 신규. 컬럼: `schoolCode/schoolName/regionCode/regionName/foundationType/schoolKind/roadAddress`.
- `NeisSyncService` 가 `schoolInfo` API 만 호출, 1,000건 페이징 × 최대 20회 = 2만 학교 상한.
- 첫 부팅 시 1회 적재 → 이후 부팅에선 캐시 존재 시 스킵(`SKIPPED_CACHED_*`). 강제 갱신은 `/api/ingest/neis/run`.
- 추후 학교별 내신 보정 가중치를 위한 기초 마스터로만 사용. 학사 일정·급식 등 다른 NEIS API 의도적 제외.
- **Why 1회 적재 후 캐시 우선:** 학교 마스터는 연 단위 변동. 매 부팅마다 2만건 호출은 NEIS 호출수 한도 낭비.

### 2. xlsx 정책 = 100% Manual ✅
- 자동 스케줄러 추가 금지. `data-staging/README.md` 에 정책 명문화.
- 운영자 흐름: ① 새 xlsx 배치 → ② `python _convert_xlsx_to_csv.py` 수동 1회 → ③ `POST /api/ingest/staging/reload` 또는 앱 재시작.
- **Why 자동화 거부:** 입결은 연 1회 변동. 파일 감시 인프라(watchdog) 추가 비용 > 1년 1회 수동 비용.

### 3. 흔적 파일 청정화 ✅
- `data-staging/_inspect_xlsx.py`, `_inspect_result.txt` 완전 삭제.
- `_convert_xlsx_to_csv.py` 는 운영자가 xlsx 갱신 시 재실행할 도구이므로 유지.

## 다음 마일스톤도 같이 적재

### 4. 해외 대학 크롤러 고도화 — `UsaScorecardSyncService`
- US Department of Education **College Scorecard API** 자동 호출.
- 환경변수 `SCORECARD_API_KEY` 있으면 `SCORECARD_SCHOOL_IDS` (IPEDS UNITID 콤마) 풀로 부팅+주간 cron 갱신.
- 기본 풀: Harvard/MIT/Stanford/Princeton/Yale/Columbia/UChicago/UC Berkeley/UCLA/NYU 10개교 UNITID 사전 배치.
- 키 없으면 `SKIPPED_NO_KEY` → `data-staging/us_cds_stats.csv` Fallback 그대로 사용.
- USA만 자동, 비-USA(UK/JAPAN/CHINA)는 staging CSV로 계속 관리.
- **Why 전 세계 자동 크롤러 안 만든 것:** 학교마다 CDS 페이지 포맷 상이, 일반화 비용 > 가치. USA 자동화로 양적 기여, 나머지는 staging CSV로 명확한 분담.
- 신규 엔드포인트:
  - `POST /api/ingest/scorecard/run` — 즉시 동기화
  - `POST /api/ingest/staging/reload` — staging UPSERT 재실행
  - `GET /api/ingest/status` — KCUE/NEIS/Scorecard 3개 상태 한 번에

### 5. 분포 시각화 강화 — `DistributionChart`
- **표본 부족 마스킹 (< 50건):** 차트 대신 점선 박스 + "노이즈가 신호보다 큽니다 (X건 더 필요)" 메시지 + 내 위치만 카드로.
- **본인 막대:** 짙은 네이비 채움 + 빨간 점선 ReferenceLine + "나" 라벨 굵게 (3중 강조).
- 각 막대 상단에 카운트 숫자 표시(LabelList).
- 축 라벨 "백분위 구간" 추가, 호버 시 cursor 음영.
- **Why 50건 임계:** 50건 이하면 5단위 20개 빈에 각 2.5건 미만 평균 → 분포라기보다 점 산포. 사용자 오해 소지 큼.
